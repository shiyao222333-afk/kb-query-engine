## 一、技术选型决策

### 方案对比

| 方案 | 优势 | 劣势 | 推荐场景 |
|------|------|------|----------|
| **Streamlit** | 10 行代码出界面，Python 原生 | 交互受限，不适合复杂布局 | 快速原型 v0.2 ✅ |
| **Gradio** | 适合 AI 应用，聊天界面开箱即用 | 定制化能力中等 | AI 对话型应用 |
| **FastAPI + Vue** | 前后端分离，扩展性强 | 开发量大，需要写前端 | 完整产品 v0.3 ✅ |
| **Flask + 原生 HTML** | 轻量，Python 全栈 | 交互体验一般 | 简单内部工具 |

### 决策结果

- **v0.2（本周可完成）**：用 **Streamlit** 做原型，快速验证交互流程
- **v0.3（1-2 个月后）**：用 **FastAPI + 原生 HTML/JS** 做完整版，保持轻量无框架依赖

## 二、v0.2 任务清单（Streamlit 原型）

  - 主区域：问答界面

### Phase 2：问答界面（1 天）

- [ ] **T2.1** 输入框 + 提交按钮
- [ ] **T2.2** 流式输出回答（Streamlit 支持 `stream` 模式）
- [ ] **T2.3** 引用标注展示（高亮 + Hover 显示原始素材）

## 三、v0.3 任务清单（完整版 FastAPI + HTML）

### Phase 1：后端 API（2-3 天）

- [ ] **T1.1** 用 FastAPI 封装 REST API：
  - `POST /api/ingest` — 摄入文档
  - `POST /api/ocr` — OCR 识别
  - `POST /api/query` — 问答查询
  - `GET /api/docs` — 知识库列表
  - `DELETE /api/docs/{source}` — 删除文档
  - `GET /api/stats` — 统计信息
- [ ] **T1.2** 添加 API Key 鉴权（简单 Bearer Token）
- [ ] **T1.3** 添加 CORS 支持（前端本地开发需要）

### Phase 2：前端界面（3-4 天）

- [ ] **T2.2** 问答界面（仿 ChatGPT 风格）：
  - 输入框 + 发送按钮
  - Markdown 渲染回答（引用标注高亮）
  - 原始素材侧边栏
- [ ] **T2.3** 知识库管理页面（Tab 切换）：
  - 上传文件（拖拽上传）
  - 文档列表 + 删除按钮
  - 统计信息卡片
- [ ] **T2.4** 设置页面：
  - LLM 配置
  - OCR 设置
  - 表格拆分阈值

### Phase 3：高级功能（2-天）

- [ ] **T3.1** 对话历史（本地 `localStorage` 存储）
- [ ] **T3.2** 多轮对话（把历史传给 LLM）
- [ ] **T3.3** 导出对话记录（Markdown / PDF）
- [ ] **T3.4** 批量摄入（拖拽多个文件）

### Phase 4：部署优化（1 天）

- [ ] **T4.1** 写 `Dockerfile`（可选，给用户多一种部署方式）
- [ ] **T4.2** 写 `start_ui.bat` / `start_ui.sh`
- [ ] **T4.3** 添加 `(知识库管理系统)` 配置（让模型感知知识库内容）
- [ ] **T4.4** 性能优化（查询结果缓存）

## 八、后记：为什么不直接用现成的？

RAGFlow / AnythingLLM 都有 Web UI，那为什么还要自己写？

1. **中文技术文档优化**：通用工具对中文 OCR + 公式 + 表格的支持不够深
2. **引用粒度控制**：表格行级拆分是我们独有的功能，通用 UI 不支持
3. **轻量级**：我们不依赖 Docker，不依赖 Node.js，纯 Python 环境就能跑
4. **可定制**：后续要加 IMA 联动、B站视频素材生成等功能，自己写的 UI 更好接入
