# Athanor · 熔知 — 问题跟踪

## v0.4.2（2026-06-17）— PATCH 修复版本

### [#I001] answer() 中 LLM 配置永远为空
- 优先级: 🔴 P0 阻断
- 状态: 🔵 Fixed
- 发现日期: 2026-06-17
- 版本: v0.4.2
- 影响: 勾选"使用 AI 回答"后始终显示"未配置 LLM API"，无法生成回答和 HTML 报告
- 根因: answer() 使用模块级常量 LLM_API_KEY，该常量在 import kb_query 时就已求值（此时 .env 未加载）
- 修复: answer() 和 auto_classify() 改为 os.environ.get("KB_LLM_API_KEY") or LLM_API_KEY

### [#I002] WebSocket 超时导致 connect lost
- 优先级: 🟡 P1 严重
- 状态: 🔵 Fixed
- 发现日期: 2026-06-17
- 版本: v0.4.2
- 影响: 搜索时出现"connect lost"，然后恢复，显示"无回答"
- 根因: NiceGUI 默认 reconnect_timeout=3.0s，但 LLM API 调用需 10-30s
- 修复: ui.run() 设置 reconnect_timeout=120

### [#I003] 系统状态一直显示离线
- 优先级: 🟡 P1 严重
- 状态: 🔵 Fixed
- 发现日期: 2026-06-17
- 版本: v0.4.2
- 影响: 页面顶部始终显示"离线"徽章，用户无法判断系统是否正常
- 根因: (1) QDRANT_URL 使用 "localhost" 解析为 IPv6 ::1，Qdrant 只监听 IPv4 127.0.0.1；(2) 状态徽章 hardcode 为"离线"
- 修复: QDRANT_URL 改为 127.0.0.1；状态徽章动态初始化 + 每 10 秒定时刷新

### [#I004] 报告链接 file:// 浏览器安全阻止
- 优先级: 🟢 P2 一般
- 状态: 🔵 Fixed
- 发现日期: 2026-06-17
- 版本: v0.4.2
- 影响: 点击"在浏览器打开"报告后提示失去连接
- 根因: 按钮用 `file://D:\...` 协议打开本地文件，浏览器安全策略阻止
- 修复: 新增 `/reports/{filename}` FileResponse 路由，按钮改为 `/reports/文件名.html`

### [#I005] 端口冲突反复出现
- 优先级: 🟡 P1 严重
- 状态: 🔵 Fixed
- 发现日期: 2026-06-17
- 版本: v0.4.2
- 影响: 每次重启服务都提示 winderror 10048 端口占用
- 根因: 旧进程未杀死，旧连接的 TIME_WAIT 也短暂占用端口
- 修复: 更新 run.bat 为自动杀旧进程再启动的脚本

### [#I006] ocr_image() 公共入口函数缺失
- 优先级: 🔴 P0 阻断
- 状态: 🔵 Fixed
- 发现日期: 2026-06-17
- 版本: v0.4.2
- 影响: 外部调用 ocr_image() 失败
- 根因: 只有私有函数 _ocr_paddle / _ocr_structured，没有公共包装
- 修复: 创建 ocr_image() 公共函数，PPStructureV3 优先，PaddleOCR 回退

### [#I007] search() 返回字段名不匹配
- 优先级: 🔴 P0 阻断
- 状态: 🔵 Fixed
- 发现日期: 2026-06-17
- 版本: v0.4.2
- 影响: main.py 搜索后无法正确显示结果
- 根因: main.py 中使用 "results" / "highlights" 字段名，实际 kb_query.search() 返回 "chunks" / "synthesis"
- 修复: main.py 改为正确的字段名

### [#I008] ingest() 参数错误
- 优先级: 🔴 P0 阻断
- 状态: 🔵 Fixed
- 发现日期: 2026-06-17
- 版本: v0.4.2
- 影响: 文本导入到 Qdrant 后内容为空或错乱
- 根因: ingest() 调用时把 text 内容当 file_path 位置参数传入
- 修复: 改为 `text=ingest_content` 关键字参数
