## 零、测试前准备

### 0.1 启动系统

```bash
cd D:\knowledge-forge
python -m streamlit run app.py
```

浏览器打开 http://localhost:8501 → 确认页面正常加载、无报错。

### 0.2 准备测试文件

在 `D:\knowledge-forge\` 下创建 `test_files\` 文件夹，依次创建以下测试文件：

| # | 文件名 | 用途 | 创建方法 |
|---|--------|------|---------|
| 1 | `hello.txt` | UTF-8 纯文本检测 | 记事本新建，输入 "Hello World 你好世界"，保存为 UTF-8 |
| 2 | `gbk_test.txt` | GBK 编码检测 | 记事本新建，输入 "这是GBK编码的测试文件"，另存为 → 编码选 ANSI（即 GBK） |

| 6 | `test.csv` | CSV 文件提取 | `姓名,年龄,城市\n张三,30,北京\n李四,25,上海` |
| 7 | `test.srt` | SRT 字幕提取 | 写入标准 SRT 格式字幕（含时间戳） |

依次上传以下扩展名的有效文件，确认每个都能正确识别：

- [ ] `.txt` → tier 2, format "text"
- [ ] `.md` → tier 2, format "text"
- [ ] `.json` → tier 2, format "text"
- [ ] `.csv` → tier 2, format "text"
- [ ] `.srt` → tier 2, format "text"
- [ ] `.html` / `.htm` → tier 1, has_auto_metadata: True
- [ ] `.pdf` → tier 1, has_auto_metadata: True
- [ ] `.epub` → tier 1, has_auto_metadata: True
- [ ] `.docx` → tier 2, format "docx"
- [ ] `.pptx` → tier 2, format "pptx"
- [ ] `.png` → tier 3（引导到 OCR Tab）
- [ ] `.jpg` / `.jpeg` → tier 3
- [ ] `.bmp` → tier 3
- [ ] `.webp` → tier 3
- [ ] `.tiff` → tier 3

## 附：常见问题排查

| 现象 | 可能原因 | 解决 |
|------|---------|------|
| 页面启动报错 | 依赖未安装 | `pip install -r requirements.txt` |
| OCR 不可用 | PaddleOCR 未装 | 检查 PaddleOCR 安装 |
| Qdrant 连接失败 | 未启动向量库 | 运行 `run.bat` |
| "自动分析" 灰色 | 未配 LLM API Key | 引擎配置页设置 `KB_LLM_API_KEY` |
| 中文乱码 | 编码检测失效 | 检查 chardet 是否正确安装 |
| `st.stop()` 报错 | Streamlit 版本 | 确保 Streamlit ≥ 1.28 |
